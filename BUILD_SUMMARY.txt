SYSTEM SPECIFICATIONS
================================================================================

PERFORMANCE CHARACTERISTICS
- Embedding Generation: 100ms per query
- Vector Search: <100ms (10k chunks)
- LLM Inference: 5-30 seconds (Mistral 7B)
- End-to-End Query: 10-50 seconds
- Single Server Capacity: ~10 concurrent queries
- Document Support: ~10,000 documents
- Total Chunks: ~100,000 supported
- Memory: 4GB minimum, 8GB recommended

API ENDPOINTS
- POST /upload - Upload and index documents
- POST /query - Question answering with sources
- GET /health - Service health check
- GET /collection-info - Knowledge base statistics
- DELETE /collection - Clear all documents
- POST /evaluate - Answer quality metrics

SUPPORTED FILE FORMATS
- PDF (via PyPDF2)
- DOCX (via python-docx)
- TXT (Plain text)
- Maximum size: 100MB per file

EMBEDDING MODEL
- Name: all-MiniLM-L6-v2
- Size: 110MB (33M parameters)
- Dimensions: 384-vector
- Languages: 128 languages supported
- Performance: ~100 docs/sec

LLM MODEL
- Provider: Ollama
- Model: Mistral 7B (default)
- Parameters: 7 billion
- Alternatives: Llama2, Neural Chat, Orca-Mini

================================================================================
TECHNOLOGY JUSTIFICATION
================================================================================

FASTAPI (Backend)
Why: Type-safe, async-first, auto-documentation, high performance
Alternatives Considered: Flask (sync), Django (heavyweight)

STREAMLIT (Frontend)
Why: Rapid development, data-app focus, easy deployment
Alternatives Considered: React (verbose), Vue (more setup)

CHROMADB (Vector Store)
Why: Lightweight, embedded, persistent, no separate server
Alternatives Considered: Weaviate (heavier), Pinecone (cloud)

SENTENCE-TRANSFORMERS (Embeddings)
Why: Semantic-aware, multilingual, lightweight, production-proven
Alternatives Considered: OpenAI Embeddings (cost), Cohere (API)

OLLAMA + MISTRAL (LLM)
Why: Open-source, local inference, data privacy, cost-effective
Alternatives Considered: GPT-4 (cost), Claude (API), self-hosted (complexity)

PYPDF2 + PYTHON-DOCX (Document Parsing)
Why: Lightweight, Python-native, no external dependencies
Alternatives Considered: pdfplumber (heavier), python-pptx (more setup)

QUICK START (5 MINUTES)
================================================================================

1. PREREQUISITES
   - Python 3.10+
   - Ollama installed and running
   - Mistral model downloaded: `ollama pull mistral`

2. SETUP
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt

3. START SERVICES
   Terminal 1: ollama serve
   Terminal 2: cd rag_system/backend && python main.py
   Terminal 3: cd rag_system/frontend && streamlit run app.py

4. ACCESS
   - UI: http://localhost:8501
   - API Docs: http://localhost:8000/docs
   - API: http://localhost:8000
